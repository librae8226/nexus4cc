# Nexus Internationalization (i18n) Design

**Date:** 2026-03-29
**Status:** Approved
**Scope:** Full frontend string extraction + multi-language support + General Settings panel

---

## Problem

All user-visible strings (~200+) in Nexus are hardcoded in Chinese. There is no i18n infrastructure. The app must support multiple languages, default to English, and allow users to switch language from a settings panel.

---

## Goals

1. Extract all hardcoded strings into translation files
2. Support 8 languages at launch (English default)
3. Add a General Settings panel with language + theme controls
4. Detect browser language on first visit; persist user preference in localStorage
5. All keys follow a consistent naming convention

---

## Non-Goals

- Server-side i18n (backend error messages stay English)
- RTL language support (Arabic, Hebrew)
- Automatic machine translation quality review (translations are best-effort at launch)

---

## Library

**`i18next` + `react-i18next` + `i18next-browser-languagedetector`**

- Industry-standard, excellent TypeScript support
- Lazy-loadable translation files per locale
- Browser language auto-detection on first visit
- Fallback chain: selected lang → `en` → key name

Install:
```
npm install i18next react-i18next i18next-browser-languagedetector
```

## File Structure

```
frontend/src/
  i18n/
    index.ts                      # i18next init, language detector setup
  locales/
    en/translation.json
    zh-CN/translation.json
    ja/translation.json
    ko/translation.json
    es/translation.json
    fr/translation.json
    de/translation.json
    ru/translation.json
```

Single namespace `translation`. Keys grouped by domain:

```
common.*          — shared: cancel, create, loading, save, rename, close, delete, edit, confirm
login.*           — login screen strings
workspace.*       — WorkspaceSelector dialog
newWindow.*       — NewWindowDialog
toolbar.*         — Toolbar component UI (panel titles, buttons)
toolbarKeys.*     — key descriptions in toolbarDefaults.ts
settings.*        — GeneralSettings panel
apiConfig.*       — SessionManager (API config management)
sessionMgr.*      — SessionManagerV2
tasks.*           — TaskPanel
files.*           — FilePanel
windowStatus.*    — windowStatus.ts status strings
tabBar.*          — TabBar tooltips/labels
```

### Key Naming Convention

- **camelCase** for all keys
- **Namespace prefix** required in component: `t('common.cancel')`
- **Dynamic values** use i18next interpolation: `t('files.deleteConfirm', { filename })`
- **Plural forms** use i18next plural suffix: `t('files.deleteAllConfirm', { count })`

## i18n Init (`frontend/src/i18n/index.ts`)

```typescript
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import LanguageDetector from 'i18next-browser-languagedetector'

import en from '../locales/en/translation.json'
import zhCN from '../locales/zh-CN/translation.json'
// ... other locales

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: { en: { translation: en }, 'zh-CN': { translation: zhCN }, ... },
    fallbackLng: 'en',
    supportedLngs: ['en', 'zh-CN', 'ja', 'ko', 'es', 'fr', 'de', 'ru'],
    detection: {
      order: ['localStorage', 'navigator'],
      caches: ['localStorage'],
      lookupLocalStorage: 'nexus_language',
    },
    interpolation: { escapeValue: false },
  })

export default i18n
```

Import `i18n/index.ts` in `main.tsx` before rendering.

## General Settings Panel

New component: `frontend/src/GeneralSettings.tsx`

Replaces the old "配置/Settings" toolbar button as the primary settings entry point.

### Sections

**1. Appearance**
- Language: `<select>` dropdown with all 8 languages listed by native name
  - On change: call `i18n.changeLanguage(code)` + save to `localStorage('nexus_language')`
- Theme: Dark / Light toggle (moved here from toolbar; toolbar keeps quick-toggle icon for convenience)

**2. API Config Profiles**
- Button "Manage API Profiles" → opens `SessionManager` overlay (existing component, unchanged)
- Brief description of what profiles are

### Triggering

- Toolbar "Settings" icon → opens `GeneralSettings` (replaces direct open of `SessionManager`)
- `GeneralSettings` renders as a modal overlay (same pattern as other panels)

### `windowStatus.ts` handling

`getWindowStatus()` currently returns Chinese strings directly. Change to return status keys (`'running'`, `'waitingInput'`, `'shellExited'`, `'disconnected'`). Call site translates: `t('windowStatus.' + status)`.

### `toolbarDefaults.ts` handling

Key `description` field currently holds Chinese strings. Change to i18n key strings (e.g., `'toolbarKeys.prevHistory'`). `Toolbar.tsx` resolves descriptions via `t(key.description)`.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Translation quality for ja/ko/es/fr/de/ru | Mark as "community translations welcome"; English fallback prevents broken UI |
| toolbarDefaults key refactor breaks toolbar layout | Test toolbar render after migration; keys are just strings |
| windowStatus key refactor breaks status dot display | STATUS_DOT_COLOR/TITLE unchanged; only string output changes |
| Bundle size increase from 8 locale files | Each locale JSON ~5-8KB; negligible. No lazy loading needed at this scale |
