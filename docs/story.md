## 当 AI 遇上碎片化时代

我的工作杂：写代码、管公司、看项目、做投资决策。这三个角色看似不同，但有一个共同点：

**最好的想法，从来不在办公桌前产生。**

- 技术架构的灵感，往往在通勤路上突然闪现

- 商业模式的漏洞，常在和创始人聊天时发现

- 运营策略的优化，可能在深夜躺床上时想通

- 投资决策的关键数据，需要在会议间隙快速验证

Claude Code 是目前最强大的 AI 助手。它能读懂代码、分析数据、生成报告、验证假设。但它有一个致命的限制：

**它活在终端里，而终端活在你的笔记本上。**

这意味着：

- 出租车上想验证一个技术方案？抱歉，笔记本在办公室

- 咖啡厅里想跑个财务模型？不好意思，电脑没带

我们生活在一个时间高度碎片化的时代。无论你是工程师、创业者、投资人，还是产品经理、数据分析师、运营总监——只要你的工作需要电脑，你就会遇到同样的困境：

**灵感转瞬即逝，但工具却被锁在某个固定的地方。**

AI 时代的生产力工具，不应该被束缚在一台设备上。

## 一个周末 Hack 的诞生

某个周末，我又一次在机场因为无法访问家里电脑上的 Claude Code 而错过了一个灵感。我想：「为什么不能让浏览器直接连到我的终端呢？」

于是我开始写代码。

第一版很简单：一个 WebSocket 服务器，连接到 tmux 会话，让我能在 iPhone 上看到终端输出。能用，但体验糟糕——手机键盘输入命令简直是噩梦。

然后我加了软键盘工具栏，把常用的快捷键（Ctrl+C、Tab、Esc）做成按钮。好多了，但还是不够顺滑。

接着我实现了滑动手势切换窗口、双指缩放调整字体大小、拖拽上传文件……一点点地，它变成了一个真正为移动端设计的终端界面。

最关键的突破是任务面板。我意识到，在移动场景下，你不需要一直盯着终端——你需要的是「发射后不管」的能力：

**下达指令，关闭手机，该干嘛干嘛。AI 在后台持续工作，随时回来查看进度。**

有意思的是，**Nexus 本身，就是在 Nexus 中开发出来的。**

我在地铁上用手机让 Claude 实现新功能，在咖啡厅里调试代码，在会议间隙修复 Bug。这个工具在解决自己的问题的同时，也在证明自己的价值。

更有意思的是，因为它是开源的，任何使用者都可以在使用过程中不断打磨迭代，然后把改进分享给社区。**每个用户既是使用者，也可以是创造者。**

不知不觉间，这个周末项目变成了我 AI 工作流的超级入口。现在，我把它开源了。

### 2. 移动端体验，专为触控优化

![移动端触控体验](https://cdn.gooo.ai/gen-images/cbfdc7145911364cd5e892a09c3e68e8f1fdab82b7d5794330d29658cb89963a.jpg)

大多数终端模拟器在手机上简直是灾难——小小的屏幕上密密麻麻的文字，想点击某个位置需要放大再放大，输入命令要在虚拟键盘上一个字母一个字母地敲……

Nexus4CC 从第一天起就是为移动端设计的：

- **滑动切换窗口**：左右滑动，在不同的 Claude Code 会话间切换，就像刷抖音一样流畅

- **双指缩放**：字体太小？两根手指一捏，瞬间放大到舒适的大小

- **软键盘工具栏**：Ctrl+C、Tab、Esc、方向键……所有常用快捷键都是一键触达

- **智能文件浏览器**：不用在终端里敲 `ls` 和 `cd`，直接点击目录树，拖拽上传文件

这不是一个「能用」的移动终端，而是一个「好用」的移动终端。

Nexus4CC 的架构天然免疫这些问题：

这意味着：

- 甚至服务器重启了？没关系，tmux 会话依然在，所有历史记录完好无损

## 更多你可能没想到的亮点

### 多项目并行，井然有序

你可能同时在做好几个项目：

- 一个客户的紧急需求

- 自己的开源项目

- 公司的主线任务

Nexus4CC 的 Project-Channel 架构让你能够优雅地管理多个项目：

- 每个项目是一个独立的 tmux session

- 每个项目下可以有多个 Channel（窗口），分别运行不同的任务

- 在手机上轻松切换项目，就像切换 App 一样简单

### 任务模板，一键启动（规划中）

有些任务你会反复执行：

- 构建 → 测试 → 部署

- 拉取最新代码 → 安装依赖 → 启动开发服务器

- 数据备份 → 清理日志 → 重启服务

Nexus4CC 支持任务模板。把这些流程保存下来，下次只需要点一个按钮，AI 就会按照预设的步骤依次执行。

**把重复劳动交给 AI，把创造力留给自己。**

### PWA 支持，像 App 一样使用

Nexus4CC 是一个渐进式 Web 应用（PWA）。你可以把它「安装」到手机主屏幕，就像一个原生 App 一样：

- 全屏体验，没有浏览器地址栏的干扰

- 离线缓存，秒开无等待

### 安全可控，数据在你手中

Nexus4CC 不是一个云服务，而是一个自托管工具。

- 所有数据都在你的电脑上

- 所有代码都在你的控制下

- 没有第三方能看到你的项目

- 没有月费，没有订阅，一次部署永久使用

通过 Cloudflare Tunnel 或 Tailscale，你可以安全地从任何地方访问，无需暴露端口，无需担心安全问题。

## 开始你的 AI 军团之旅

![5步部署流程](https://cdn.gooo.ai/gen-images/254cf728a36ad8ba144bbcc8b1f4156eef9583345c444784f2919ce6f1186676.jpg)

Nexus4CC 是完全开源的，MIT 许可证，你可以自由使用、修改和分发。

**项目地址**：<https://github.com/librae8226/nexus4cc>

**五分钟快速部署**：

```bash
# 1. 克隆项目
git clone https://github.com/librae8226/nexus4cc.git
cd nexus4cc

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，设置密码和工作目录

# 3. 安装依赖并构建
npm install
cd frontend && npm install && npm run build && cd ..

# 4. 启动服务
npm start

# 5. 在任何设备上打开 http://你的服务器IP:3000
```

就这么简单。
